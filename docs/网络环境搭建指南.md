### 网段划分

| 网段名称 | CIDR | 安全级别 | 虚拟网络适配器 | 说明 |
|---------|------|---------|--------------|------|
| DMZ | 192.168.100.0/24 | **低安全防御区** | VMnet2 / vboxnet1 | 对外暴露服务，受限访问内网 |
| Internal | 192.168.200.0/24 | **高安全防御区** | VMnet3 / vboxnet2 | 核心业务与办公网络 |
| Management | 192.168.10.0/24 | **安全监测区** | VMnet4 / vboxnet3 | SOC 节点隔离，仅接收日志 |

## 二、访问控制规则

### 2.1 默认策略

| 区域 | 默认入站 | 默认出站 |
|------|---------|---------|
| DMZ | DROP | DROP |
| Internal | DROP | DROP |
| Management | DROP | DROP（仅允许到 DMZ/Internal 的 SSH/WinRM 出向） |

### 2.2 详细规则矩阵

```
规则格式：源 → 目标 : 端口/协议 (用途)

【外部 → DMZ】
  internet → web-dmz : 80/tcp, 443/tcp      (正常 Web 访问)
  attacker → web-dmz : ANY                    (攻击模拟，不限制)

【DMZ → Internal】
  web-dmz → db-internal : 3306/tcp            (MySQL 业务查询)

【DMZ → Management】
  ALL BLOCKED                                  (DMZ 不可触碰监测区)

【Internal 内部】
  dc-ad → ALL Internal : 88, 389, 445, 135    (AD 域控服务)
  win-workspace → dc-ad : 88, 389             (域认证)
  Internal 同网段互访 : ALLOW                  (但需经过 dc-ad 认证)

【Internal → Management】(仅日志上报)
  ALL Internal → soc-node : 5044/tcp           (Logstash / Winlogbeat)
  ALL Internal → soc-node : 9200/tcp           (Elasticsearch，可选直写)
  ALL Internal → soc-node : 5000/tcp           (Flask API，DeepSecurity 后端)
  ALL Internal → soc-node : 1514/tcp, 514/udp  (Syslog)

【Internal → DMZ】
  ALL BLOCKED                                  (内网不主动连 DMZ)

【Internal → 外部】
  ALL BLOCKED                                  (无互联网出口)

【Management → DMZ / Internal】(管理出向)
  soc-node → DMZ : 22/tcp                      (SSH 巡检)
  soc-node → Internal : 22/tcp, 5985/tcp       (SSH + WinRM 巡检)
  soc-node → Internal : 3306/tcp               (MySQL 监控采集)
```

## 三、每台主机详细配置

### 3.1 attacker — 外部攻击机

| 配置项 | 值 |
|--------|-----|
| OS | Kali Linux 2024+ |
| IP | DHCP (NAT 模式) |
| 本地账号 | `kali` (sudo 组) |
| 预装工具 | nmap, Metasploit, Hydra, impacket, crackmapexec, sqlmap, Burp Suite Community |
| 日志 | 无需开启（攻击机不在监测范围内） |
| 防火墙 | 无需配置（NAT 出向） |

### 3.2 web-dmz — DMZ Web/API 服务器

| 配置项 | 值 |
|--------|-----|
| OS | Ubuntu 22.04 LTS |
| IP | 192.168.100.10 |
| 本地账号 | `admin` (sudo 组，用于运维) |
| 安装服务 | Nginx, PHP-FPM (可选), Python3 + Flask (API 模拟) |
| 业务端口 | 80/tcp (HTTP), 443/tcp (HTTPS) |

**日志配置：**

| 日志类型 | 路径/来源 | 转发目标 |
|---------|----------|---------|
| Nginx Access Log | `/var/log/nginx/access.log` | soc-node:5044 (Logstash) |
| Nginx Error Log | `/var/log/nginx/error.log` | soc-node:5044 |
| Auth Log | `/var/log/auth.log` | soc-node:514 (Syslog) |
| Auditd 系统调用日志 | `/var/log/audit/audit.log` | soc-node:5044 |
| Syslog | `/var/log/syslog` | soc-node:514 |

**防火墙规则（iptables）：**

```bash
# 默认策略
iptables -P INPUT DROP
iptables -P OUTPUT DROP
iptables -P FORWARD DROP

# 允许 loopback
iptables -A INPUT -i lo -j ACCEPT
iptables -A OUTPUT -o lo -j ACCEPT

# 允许对外 HTTP/HTTPS 已建立连接
iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT
iptables -A OUTPUT -m state --state ESTABLISHED,RELATED -j ACCEPT

# 入站：允许 HTTP/HTTPS（来自任意源，模拟公网暴露）
iptables -A INPUT -p tcp --dport 80 -j ACCEPT
iptables -A INPUT -p tcp --dport 443 -j ACCEPT

# 入站：允许 SSH（仅 soc-node 管理网段）
iptables -A INPUT -s 192.168.10.0/24 -p tcp --dport 22 -j ACCEPT

# 出站：仅允许到 db-internal 的 MySQL
iptables -A OUTPUT -d 192.168.200.20 -p tcp --dport 3306 -j ACCEPT

# 出站：允许 DNS 和 NTP
iptables -A OUTPUT -p udp --dport 53 -j ACCEPT
iptables -A OUTPUT -p udp --dport 123 -j ACCEPT

# 出站：允许日志转发到 soc-node
iptables -A OUTPUT -d 192.168.10.10 -p tcp --dport 5044 -j ACCEPT
iptables -A OUTPUT -d 192.168.10.10 -p udp --dport 514 -j ACCEPT
```

| Auditd 系统调用日志 | `/var/log/audit/audit.log` | soc-node:5044 |

**防火墙规则（iptables）：**

```bash
# 默认策略
iptables -P INPUT DROP
iptables -P OUTPUT DROP
iptables -P FORWARD DROP

# loopback
iptables -A INPUT -i lo -j ACCEPT
iptables -A OUTPUT -o lo -j ACCEPT

# 已建立连接
iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT
iptables -A OUTPUT -m state --state ESTABLISHED,RELATED -j ACCEPT

# 入站：允许来自 web-dmz 的 MySQL
iptables -A INPUT -s 192.168.100.10 -p tcp --dport 3306 -j ACCEPT

# 入站：允许来自 Internal 网段的 SQL Server
iptables -A INPUT -s 192.168.200.0/24 -p tcp --dport 1433 -j ACCEPT

# 入站：允许 SSH（仅 soc-node 和内网管理）
iptables -A INPUT -s 192.168.10.0/24 -p tcp --dport 22 -j ACCEPT
iptables -A INPUT -s 192.168.200.0/24 -p tcp --dport 22 -j ACCEPT

# 出站：DNS/NTP
iptables -A OUTPUT -p udp --dport 53 -j ACCEPT
iptables -A OUTPUT -p udp --dport 123 -j ACCEPT

# 出站：日志转发到 soc-node
iptables -A OUTPUT -d 192.168.10.10 -p tcp --dport 5044 -j ACCEPT
iptables -A OUTPUT -d 192.168.10.10 -p udp --dport 514 -j ACCEPT

# 出站：AD 域认证到 dc-ad（如果加入域）
iptables -A OUTPUT -d 192.168.200.10 -p tcp -m multiport --dports 88,389 -j ACCEPT
```

| 配置项 | 值 |
|--------|-----|

**日志配置：**

|---------|----------------------|---------|
| 安全日志 (登录/登出) | `Security` (Event ID 4624/4625/4672 等) | soc-node:5044 (Winlogbeat) |

| 应用程序日志 | `Application` | soc-node:5044 |
| Sysmon 日志 | `Microsoft-Windows-Sysmon/Operational` | soc-node:5044 |
| AD DS 审计 | `Directory Service` | soc-node:5044 |
| PowerShell 日志 | `Microsoft-Windows-PowerShell/Operational` | soc-node:5044 |

**防火墙规则（Windows 高级防火墙）：**

```powershell
# 默认策略
Set-NetFirewallProfile -Profile Domain,Public,Private -DefaultInboundAction Block
Set-NetFirewallProfile -Profile Domain,Public,Private -DefaultOutboundAction Block

# 入站：AD DS 服务（仅 Internal 网段）
New-NetFirewallRule -Name "AD_Kerberos" -Direction Inbound -RemoteAddress 192.168.200.0/24 -LocalPort 88 -Protocol TCP -Action Allow
New-NetFirewallRule -Name "AD_LDAP" -Direction Inbound -RemoteAddress 192.168.200.0/24 -LocalPort 389 -Protocol TCP -Action Allow
New-NetFirewallRule -Name "AD_SMB" -Direction Inbound -RemoteAddress 192.168.200.0/24 -LocalPort 445 -Protocol TCP -Action Allow
New-NetFirewallRule -Name "AD_RPC" -Direction Inbound -RemoteAddress 192.168.200.0/24 -LocalPort 135 -Protocol TCP -Action Allow

# 入站：RDP（仅 soc-node 管理）
New-NetFirewallRule -Name "RDP_Mgmt" -Direction Inbound -RemoteAddress 192.168.10.10 -LocalPort 3389 -Protocol TCP -Action Allow

# 入站：WinRM（仅 soc-node 管理）
New-NetFirewallRule -Name "WinRM_Mgmt" -Direction Inbound -RemoteAddress 192.168.10.10 -LocalPort 5985 -Protocol TCP -Action Allow

# 出站：日志转发到 soc-node
New-NetFirewallRule -Name "Logstash_Out" -Direction Outbound -RemoteAddress 192.168.10.10 -LocalPort 5044 -Protocol TCP -Action Allow

# 出站：DNS
New-NetFirewallRule -Name "DNS_Out" -Direction Outbound -Protocol UDP -LocalPort 53 -Action Allow
```

## 四、部署方案

### 方案 A：VirtualBox 手动部署（推荐，免费跨平台）

#### 步骤 1：创建虚拟网络

打开 VirtualBox → 管理 → 工具 → 网络管理器 → 仅主机网络：

| 网络名称 | IPv4 地址 | 掩码 | DHCP |
|---------|----------|------|------|
| vboxnet1 (DMZ) | 192.168.100.1 | 255.255.255.0 | 禁用 |
| vboxnet2 (Internal) | 192.168.200.1 | 255.255.255.0 | 禁用 |
| vboxnet3 (Management) | 192.168.10.1 | 255.255.255.0 | 禁用 |

#### 步骤 2：创建虚拟机

按"三、主机清单"中的规格创建 6 台 VM，并分配网络适配器：

| VM | 适配器 1 | 适配器 2 |
|----|---------|---------|
| attacker | NAT | — |
| web-dmz | vboxnet1 (DMZ) | — |
| db-internal | vboxnet2 (Internal) | — |
| dc-ad | vboxnet2 (Internal) | — |
| win-workspace | vboxnet2 (Internal) | — |
| soc-node | vboxnet3 (Management) | vboxnet2 (Internal) — 双网卡 |

> **soc-node 双网卡说明**：Management 网卡用于接收日志，Internal 网卡用于主动巡检和 Elasticsearch 暴露。该 VM 分配 IP 192.168.10.10 (Management) + 192.168.200.50 (Internal)。

#### 步骤 3：安装操作系统

- Linux VM：安装 Ubuntu Server 22.04 LTS，安装时设置静态 IP（见主机清单）。
- Windows VM：安装 Windows Server 2022 / Windows 10，安装后设置静态 IP。

#### 步骤 4：执行各主机初始化脚本

将 `infra/lab/scripts/` 目录中的脚本复制到对应 VM 并执行：

```bash
# Linux 主机（web-dmz / db-internal / soc-node）
sudo bash setup-web-dmz.sh      # 在 web-dmz 上执行
sudo bash setup-db-internal.sh  # 在 db-internal 上执行
sudo bash setup-soc.sh          # 在 soc-node 上执行
```

```powershell
# Windows 主机（dc-ad / win-workspace）
# 以管理员身份运行 PowerShell
.\setup-dc.ps1
.\setup-workspace.ps1
```

#### 步骤 5：验证网络联通性

```bash
# 在 attacker 上：验证可访问 DMZ
nmap -sV 192.168.100.10

# 在 web-dmz 上：验证可访问 db-internal:3306
mysql -h 192.168.200.20 -u <DB_USER> -p

# 在 web-dmz 上：验证不可访问 dc-ad
ping 192.168.200.10           # 应超时
nmap -p 445 192.168.200.10    # 应 filtered

# 在 soc-node 上：验证可 SSH 到各 Linux 主机
ssh admin@192.168.100.10
ssh dba@192.168.200.20

# 在 soc-node 上：验证收到日志
curl -s http://localhost:9200/_cat/indices
```

### 方案 B：Vagrant 一键部署

如果你已安装 Vagrant + VirtualBox，使用 `infra/lab/Vagrantfile`：

```powershell
cd infra\lab
vagrant up
```

Vagrant 将自动创建 6 台 VM、配置网络并执行 provisioning 脚本。首次 `up` 约需 20-40 分钟（取决于网络和磁盘速度）。

启动后：
```powershell
vagrant ssh web-dmz     # SSH 到 DMZ Web 服务器
vagrant ssh soc-node    # SSH 到 SOC 节点
vagrant rdp dc-ad       # RDP 到域控（仅 VirtualBox provider）
vagrant halt            # 关闭所有 VM
vagrant destroy -f      # 删除所有 VM
```

> **注意**：Vagrant 方案需要下载各 OS 的 Vagrant Box（约 5-10 GB），请确保网络通畅。Windows Box 通常需要额外配置许可证。

### 方案 C：Docker Compose 快速演示

如果不便运行多台 VM，可使用 Docker 方案快速验证服务间联通性和日志采集：

```powershell
cd infra\lab
docker compose up -d
```

Docker 方案使用以下映射：
- 3 个 Docker 网络对应 DMZ/Internal/Management
- Ubuntu 容器模拟 web-dmz、db-internal、soc-node
- Windows 容器（如可用）或额外 Linux 容器模拟 dc-ad 和 win-workspace 的服务端口
- Kali 容器模拟 attacker

> **限制**：Docker 方案无法完整模拟 Windows 域控、Sysmon、Windows Event Log 等 Windows 特有功能。仅用于快速验证网络拓扑和日志采集管道。如需完整靶场体验，请使用方案 A 或 B。

## 七、安全注意事项

- 所有密码使用 `<PLACEHOLDER>` 标记，实际部署时替换为强密码。
- 靶场网络为完全隔离的 Host-Only 网络，不与宿主机物理网络互通（除 attacker 的 NAT 用于下载工具）。
- 禁止在靶场中存放真实敏感数据。
- 攻击机 (Kali) 仅可在靶场内部网络发起攻击，禁止对接互联网进行未授权测试。
- 部署完成后建议导出 OVA 模板，便于快速恢复和团队共享。
